Практическое задание:
реализация с использованием управляемых форм
(версия платформы не имеет значения)

По возможности сохранить в `.dt` или `.cf`.

1. Создать справочник «Номенклатура».
Реквизиты: артикул, наименование.
Тип значения не важен.

2. Создать справочник «Характеристики».
В нем должны быть созданы 2 свойства: цвет, размер.
Справочник «Характеристики» должен быть подчинен справочнику «Номенклатура».

3. Создать любой документ, где будет табличная часть с реквизитами:
- номенклатура (тип значения из пункта 1),
- характеристика (тип значения из пункта 2),
- количество.

4. Характеристика должна быть связана с номенклатурой в поле выбора.
При выборе характеристики должны отбираться только те значения, которые принадлежат выбранной номенклатуре из пункта 1.

5. Артикул номенклатуры должен выводиться на форме в отдельной колонке табличной части.

6. Создать регистр накопления, куда должны выгружаться данные из документа при проведении:
- номенклатура,
- характеристика,
- количество.

7. Создать простой механизм контроля остатков с выводом текста ошибки.
Механизм должен отрабатывать при проведении документа.

8. Создать простой отчет, который показывает:
- начальный остаток,
- приход,
- расход,
- конечный остаток товаров.

Отчет должен быть реализован с использованием СКД.

9. Создать простой HTTP-сервис, который будет возвращать данные остатков:
- дата,
- артикул,
- номенклатура,
- характеристика,
- количество.

Формат ответа: JSON.
Отбор: по списку артикулов, который приходит в параметрах запроса.
