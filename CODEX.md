Ты — эксперт по `1С:Предприятие 8.3`.

Этот файл задает короткий обязательный протокол работы по 1С в проекте.
Подробные правила и примеры вынесены в специализированные документы, чтобы не дублировать их здесь.

## 1. Главные документы проекта

При задаче по 1С ориентируйся на них в таком порядке:
1. `1C_DOCS_MAP.md`
2. `CODEX_STRICT_1C.md`
3. `CODEX_MISTAKES_1C.md`
4. `PATTERNS_1C.md`
5. `RULES_1C_QUICK.md`
6. `RULES_1C.md`

## 2. Источники истины по ролям

- `RULES_1C_QUICK.md` — обязательное ядро для ежедневной работы.
- `RULES_1C.md` — полный стандарт для сложных и нетиповых задач.
- `CODEX_STRICT_1C.md` — жесткий локальный протокол поведения в этом проекте.
- `CODEX_MISTAKES_1C.md` — реальные ошибки и антипримеры, которые нельзя повторять.
- `PATTERNS_1C.md` — успешные локальные решения, которые можно брать как эталон.
- `PROMPT_STRICT_1C.txt` — готовая фраза для включения строгого режима в запросе.

## 3. Обязательный процесс

Для всех задач по 1С соблюдай такой порядок:
1. сначала `CODEX_MISTAKES_1C.md`;
2. потом `PATTERNS_1C.md`, если в проекте уже есть подходящий успешный пример;
3. потом локальный эталон в текущем проекте, `samples_1c` или `tools/reference_indexes`;
4. потом одна реализация без архитектурных прыжков;
5. если эталона нет, честно остановись и скажи, что нужен образец, а не сочиняй.

## 4. Жесткие запреты

Запрещено:
- выдумывать XML-структуру метаданных;
- выдумывать управляемые формы и события форм;
- выдумывать имена форм и параметры `ОткрытьФорму(...)`;
- выдумывать `cfg:CatalogRef...`, `cfg:CatalogObject...`, `cfg:Document...`, `cfg:EnumRef...`;
- выдумывать `WindowOpeningMode`, `Attributes`, `Commands`, `ChildItems`, `StandardAttributes`;
- подменять стандартное поведение формы пользовательским кодом без подтвержденного примера;
- считать смену текста ошибки решением без проверки корневой причины.

## 5. Что делать перед выдачей результата

Перед завершением задачи по 1С:
- пройди применимые чек-листы из `RULES_1C.md`;
- проверь, что не повторена ошибка из `CODEX_MISTAKES_1C.md`;
- если в задаче выявлена новая реальная ошибка, кратко добавь ее в `CODEX_MISTAKES_1C.md`;
- если решение стало локальным эталоном, добавь его в `PATTERNS_1C.md`;
- если запуск, загрузка или тест в 1С выполнял пользователь вручную, явно зафиксируй это в ответе.
